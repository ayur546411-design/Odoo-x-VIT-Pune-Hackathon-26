## 🔴 The Problem We Solve

Corporate expense reimbursement in India remains painfully broken. Employees waste productive hours filling paper forms, chasing email approvals, and manually converting currencies. Finance teams drown in spreadsheets while receipts get lost and fraud goes undetected.

**Approval Engine — Core Algorithm:**

```python
def process_approval(expense_id, approver_id, action, comment):
    expense = get_expense(expense_id)
    workflow = get_workflow(expense.company_id)

    # Record the action with full audit trail
    save_action(expense_id, approver_id, action, comment)

    if action == "REJECTED":
        expense.status = "REJECTED"       # Rejection stops entire chain
        notify_employee(expense, comment)
        return

    # Check conditional rules (can short-circuit the chain)
    rules = get_rules(workflow.id)
    for rule in rules:
        if rule.type == "SPECIFIC_APPROVER":
            if approver_id == rule.specific_approver_id:
                expense.status = "APPROVED"      # CFO override
                return

        if rule.type == "PERCENTAGE":
            approval_count = count_approvals(expense_id)
            total_approvers = count_total_approvers(workflow.id)
            if (approval_count / total_approvers) >= rule.threshold:
                expense.status = "APPROVED"       # 60% threshold met
                return

        if rule.type == "HYBRID":
            # Either condition can trigger approval
            if approver_id == rule.specific_approver_id:
                expense.status = "APPROVED"
                return
            if (approval_count / total_approvers) >= rule.threshold:
                expense.status = "APPROVED"
                return

    # No conditional rule triggered → move to next sequential step
    next_step = get_next_step(workflow.id, expense.current_step)
    if next_step:
        expense.current_step += 1
        create_approval_request(expense, next_step.approver_id)
    else:
        expense.status = "APPROVED"               # All steps complete
```

|:---|:---|

| **API Protection** | All endpoints require valid JWT bearer token |
| **Data Isolation** | Company-scoped queries — users never see other companies' data |
| **Input Validation** | Pydantic v2 schema validation on all requests |

### ⚡ Quick Start

<details>
<summary><b>🔧 1. Backend Setup (FastAPI + MongoDB)</b></summary>

```bash
# Clone the repository
git clone https://github.com/ayur546411-design/Odoo-x-VIT-Pune-Hackathon-26.git
cd Odoo-x-VIT-Pune-Hackathon-26/Website/Backend

# Create virtual environment
python -m venv venv
source venv/bin/activate          # Linux/Mac
# venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt

# Create .env file with your credentials
cat > .env << EOF
MONGO_URI=mongodb+srv://<user>:<pass>@cluster.mongodb.net/expenseflow
JWT_SECRET=your-super-secret-key-here
JWT_ALGORITHM=HS256
BREVO_API_KEY=your-brevo-api-key
EOF

# Start the server
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 🎉 Swagger docs → http://localhost:8000/docs
```

</details>

```bash
cd Odoo-x-VIT-Pune-Hackathon-26/Website/Frontend

# Install dependencies
npm install

# Configure API base URL
# Edit src/api/axios.js → update baseURL to your backend

# Start development server
npm run dev

# 🎉 Open → http://localhost:5173

# Build for production
npm run build
```

</details>

# Enable MongoDB Atlas Data API:

# Run on device/emulator
#   Min SDK: 26 (Android 8.0)

# For physical device:
